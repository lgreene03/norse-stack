# Demo Assets

This directory holds reproducible demo artifacts for the Norse Stack: a terminal
recording of the end-to-end smoke test and snapshots of the live Grafana
dashboard. Everything here is generated by
[`scripts/record-demo.sh`](../../scripts/record-demo.sh) — nothing is uploaded or
turned into a video automatically.

```bash
# from the repo root, with the stack running (make up):
./scripts/record-demo.sh            # terminal cast + Grafana PNGs
./scripts/record-demo.sh --cast     # just the asciinema cast
./scripts/record-demo.sh --grafana  # just the Grafana PNGs
```

## 1. Terminal cast (asciinema)

`record-demo.sh --cast` records [`scripts/smoke.sh`](../../scripts/smoke.sh)
running end-to-end into `docs/demo/smoke.cast` using
[asciinema](https://asciinema.org) (`brew install asciinema`).

- Play locally: `asciinema play docs/demo/smoke.cast`
- Get a shareable link: `asciinema upload docs/demo/smoke.cast`

> **Placeholder:** `smoke.cast` is intentionally not committed yet — casts are
> machine/timing-specific. Run the command above to generate one. Once uploaded,
> drop the asciinema badge here, e.g.
> `[![asciicast](https://asciinema.org/a/XXXXX.svg)](https://asciinema.org/a/XXXXX)`.

## 2. Grafana dashboard snapshots

`record-demo.sh --grafana` calls Grafana's render endpoint
(`/render/d/norse-stack-main`) and saves PNGs to `docs/demo/images/`.

> **Status on this machine:** the bundled Grafana image does **not** ship the
> `grafana-image-renderer` plugin, so the render endpoint returns a small
> "No image renderer available" placeholder rather than a real dashboard image.
> The capture script detects this (by image size) and refuses to commit the
> misleading placeholder. Two ways to get real PNGs:
>
> 1. **Enable the renderer plugin** so `record-demo.sh --grafana` works
>    automatically. Add to the `grafana` service in
>    [`docker-compose.yml`](../../docker-compose.yml):
>    ```yaml
>    grafana:
>      environment:
>        GF_INSTALL_PLUGINS: grafana-image-renderer
>    ```
>    (or run the standalone `grafana/grafana-image-renderer` container and point
>    Grafana at it via `GF_RENDERING_SERVER_URL` / `GF_RENDERING_CALLBACK_URL`).
>    Then re-run the script — a valid `images/grafana-dashboard.png` will appear.
>
> 2. **Manual screenshot.** Open <http://localhost:3001/d/norse-stack-main>
>    (login `admin` / `norse`), and save a screenshot to
>    `docs/demo/images/grafana-dashboard.png`.

The dashboard (`Norse Stack — Trading Pipeline`, uid `norse-stack-main`) has 11
panels: signal-to-decision latency (p50/p95/p99), feature event age, orders
generated, signal-to-fill latency, risk-halt status, max-drawdown gauge,
portfolio value, fills total, realized PnL, and Sleipnir order submission rate.
